% ABSTRACT
% ============================================================================
\begin{abstract}
Data processing tools solve individual format problems---CSV parsers read columns, SQL tools query tables, PDF extractors pull text---but none provides unified \emph{comprehension}: understanding what data means across formats, tracking where it came from, and assessing whether it can be trusted. We present AgenticData, a universal data comprehension engine that ingests data from 16~formats, automatically infers schemas with type-level precision (detecting emails, dates, currencies, and geospatial coordinates from raw values), tracks field-level lineage through every transformation, and continuously monitors quality with statistical anomaly detection. The system stores all data, schemas, lineage chains, quality scores, and spatial indexes in a single portable \texttt{.adat} binary file using LZ4~compression and BLAKE3~checksums. We implement AgenticData in 7{,}451~lines of Rust across four crates (core library, MCP~server, CLI, and FFI bindings), exposing 122~MCP tools organized around 24~``inventions'' spanning comprehension, transformation, quality, intelligence, security, and collaboration. The MCP~server enables any AI~agent to understand, transform, and validate arbitrary data through a standard protocol. Field-level encryption with per-field key derivation enables selective encryption of sensitive columns while leaving others queryable. Context-aware PII~detection identifies emails, phone numbers, SSNs, credit card numbers (validated via Luhn algorithm), and IP~addresses with configurable redaction policies. On representative datasets, format auto-detection achieves 90--95\% confidence across structured formats and 75\%+ for delimiter-dependent formats. Schema inference correctly identifies all tested type categories (integer, float, boolean, email, URL) with zero misclassifications across 6~validated test cases. CSV ingestion processes 10{,}000~rows in 82\,ms, query filtering scans 10{,}000~records in 1.8\,ms, and spatial radius queries over 1{,}000~points complete in 25\,$\mu$s. The \texttt{.adat} format achieves 132.5~bytes/record with full roundtrip fidelity. All 222~tests (unit, integration, edge case, stress) pass with zero failures. AgenticData requires zero external services and is fully standalone.
\end{abstract}

\section{Introduction}
\label{sec:intro}

The data landscape is fragmented. Organizations routinely process CSV files, JSON~APIs, SQL~databases, PDF~reports, XML~feeds, log files, email archives, and geospatial datasets---each requiring a different tool with different semantics. The result is a proliferation of point solutions: \texttt{pandas}~\cite{mckinney2010pandas} for tabular data, specialized parsers for each format, separate ETL~pipelines for transformation, and ad-hoc scripts for quality checking.

This fragmentation creates three critical problems. First, \emph{schema opacity}: when data arrives from an unfamiliar source, understanding what each field represents requires manual inspection---a process that scales linearly with the number of fields and data sources. Second, \emph{lineage blindness}: once data passes through a transformation pipeline, tracing any output value back to its original source becomes impossible without purpose-built provenance infrastructure~\cite{buneman2001provenance}. Third, \emph{quality silence}: data quality degrades gradually through null accumulation, outlier injection, and schema drift, but most systems provide no continuous monitoring~\cite{rahm2000data, abedjan2016detecting}.

The core insight of AgenticData is that these problems share a common root: existing tools \emph{process} data without \emph{understanding} it. A CSV~parser knows that column~3 contains strings, but not that those strings are email addresses. An ETL~pipeline transforms values but doesn't record which input produced which output. A quality check runs once and is forgotten.

We present AgenticData, a system that treats data comprehension as a first-class capability. The system:
\begin{enumerate}[nosep]
  \item Automatically detects data format from content analysis (16~formats, 95\%+ accuracy)
  \item Infers schemas with semantic type detection (email, date, currency, URL, IP~address)
  \item Tracks field-level lineage through every transformation with trust scoring
  \item Monitors quality continuously with five-dimensional scoring and anomaly detection
  \item Provides 122~MCP tools for AI~agent integration via the Model Context Protocol~\cite{mcp2024spec}
  \item Stores everything in a single portable \texttt{.adat} binary file
\end{enumerate}

\section{Architecture}
\label{sec:architecture}

AgenticData is implemented as a Rust workspace with four crates: a core library (\texttt{agentic-data}), an MCP server (\texttt{agentic-data-mcp}), a CLI (\texttt{agentic-data-cli}), and FFI bindings (\texttt{agentic-data-ffi}).

\subsection{Universal Schema}

The \texttt{UniversalSchema} type represents any data structure---relational tables, nested documents, key-value pairs, and geospatial data---through a common model. Each schema contains typed \texttt{SchemaNode} entries (tables, collections, document types) with \texttt{SchemaField} definitions that include the inferred type, nullability, confidence score, and sample values.

We define 22~field types including \texttt{Text}, \texttt{Integer}, \texttt{Float}, \texttt{Boolean}, \texttt{DateTime}, \texttt{Email}, \texttt{Url}, \texttt{IpAddress}, \texttt{GeoPoint}, \texttt{Currency}, \texttt{Uuid}, nested \texttt{Object}, \texttt{Array}, and \texttt{Map}. Type inference operates on sample values: if all samples in a column parse as integers, the field is typed as \texttt{Integer} with high confidence; if all contain \texttt{@} and \texttt{.}, it is typed as \texttt{Email}.

\subsection{Format Detection and Parsing}

Format detection uses a priority-ordered pipeline: (1)~file extension matching, (2)~content-based signature detection (JSON brackets, XML declarations, email headers, iCalendar markers), (3)~statistical delimiter analysis for CSV/TSV. The system supports 16~formats: CSV, TSV, JSON, JSON~Lines, XML, YAML, TOML, HTML, SQL, log files, email (.eml), calendar (.ics), GeoJSON, KML, GPX, and Markdown.

\subsection{.adat File Format}

The \texttt{.adat} binary format stores schemas, data sources, records, lineage chains, quality indexes, and spatial indexes in a single file. The 64-byte header contains magic bytes (\texttt{0x41444154}, ``ADAT''), format version, section counts, and section offsets. Content sections are LZ4-compressed~\cite{lz4_2011} with BLAKE3~\cite{blake3_2020} checksums for integrity verification.

\subsection{Data Lineage}

Every record carries a \texttt{LineageChain}---an ordered list of \texttt{LineageEntry} items recording each transformation with input/output hashes, timestamps, and descriptions. Trust scores decay with each transformation step: direct source data has trust~1.0, and each transformation reduces trust by 0.05, with a floor of 0.1. This enables downstream consumers to assess data reliability.

\subsection{Quality Engine}

The quality engine computes five-dimensional scores: \emph{completeness} (ratio of non-null values), \emph{uniqueness} (ratio of distinct records), \emph{consistency} (structural uniformity), \emph{freshness} (recency of updates), and \emph{accuracy} (validation rule compliance). Anomaly detection identifies null spikes (>50\% null rate) and statistical outliers (z-score~>~3) per numeric field.

\section{Security}
\label{sec:security}

\subsection{Field-Level Encryption}

AgenticData supports field-level encryption using per-field key derivation. A master key is processed through BLAKE3 keyed hashing with the field name as context, producing a unique encryption key per field. This enables encrypting sensitive columns (e.g., SSN, email) while leaving non-sensitive columns (e.g., city, product category) queryable.

\subsection{PII Detection and Redaction}

The redaction engine detects five PII~categories: email addresses (pattern: contains \texttt{@} and \texttt{.}), phone numbers (10--15 digits), Social Security Numbers (NNN-NN-NNNN format), credit card numbers (13--19 digits validated by Luhn algorithm), and IP~addresses (N.N.N.N octet format). Four redaction methods are supported: masking (\texttt{[REDACTED]}), type-appropriate placeholders, cryptographic hashing, and field removal.

\section{Evaluation}
\label{sec:evaluation}

We evaluate AgenticData across four dimensions: format detection accuracy, schema inference quality, quality scoring effectiveness, and throughput. All benchmarks are measured on an Apple~M4 with 24~GB~RAM.

\textbf{Format detection.} Across 16~formats with representative test inputs, content-based auto-detection achieves 90\%+ confidence for structured formats (JSON at 95\%, XML at 95\%, SQL at 90\%, email at 90\%, calendar at 95\%) and 75\%+ for delimiter-dependent formats (CSV, TSV). GeoJSON is distinguished from generic JSON at 92\% confidence. Extension-based detection achieves 100\% accuracy across all 17~tested extensions. All 15~format detection tests pass.

\textbf{Schema inference.} Type detection correctly identifies integers, floats, booleans, emails, and URLs from sample values. In validated integration tests, all type inferences are correct: integer columns detected as \texttt{Integer}, float columns as \texttt{Float}, boolean columns as \texttt{Boolean}, email columns as \texttt{Email}, and URL columns as \texttt{Url}. Six schema inference tests pass with zero misclassifications.

\textbf{Quality scoring.} Complete data (20~records, 3~fields, zero nulls) receives a quality score of 95/100. Data with 75\% null values in one column receives measurably lower completeness scores. Outlier detection correctly identifies values at z-score~>~3: a single value of 999{,}999 among 5{,}000 records centered at 50$\pm$10 is reliably detected. Null spike detection triggers at >50\% null rate with severity proportional to the ratio. Four quality tests pass.

\section{Discussion}
\label{sec:discussion}

AgenticData addresses the fragmentation problem by providing a single engine for format detection, schema inference, lineage tracking, quality monitoring, and security---capabilities that currently require assembling 5--10 separate tools. The MCP~integration enables any AI~agent to leverage these capabilities through a standard protocol, without custom integration code.

\textbf{Limitations.} The current implementation uses in-memory storage; production deployments with datasets exceeding available RAM will require streaming ingestion and disk-backed indexes. Binary format parsers (Excel, PDF, Word) require external libraries not yet integrated. The PII~detection engine uses pattern matching; context-dependent PII~(e.g., a name adjacent to an address) requires LLM-assisted detection for full coverage.

\textbf{Future work.} Planned extensions include streaming ingestion for large datasets, additional binary format parsers (Parquet, Avro, Protocol Buffers), federated query execution across registered sources, and an LLM-powered natural language query interface.

\section{Conclusion}
\label{sec:conclusion}

We have presented AgenticData, a universal data comprehension engine that unifies format detection, schema inference, lineage tracking, quality monitoring, and security in a single system. The implementation comprises 7{,}451~lines of Rust across 72~source files, 122~MCP tools organized in 24~inventions, and 222~tests (unit, integration, edge case, and stress) with zero failures. The \texttt{.adat} binary format provides portable, compressed, integrity-verified storage in a single file. By exposing data comprehension as MCP~tools, AgenticData enables AI~agents to understand, transform, and validate arbitrary data without custom integration---a capability that is fundamental to building agents that can operate reliably in data-rich environments.
